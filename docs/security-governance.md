# Security Governance

## Purpose

Security governance defines how security decisions are made, documented, approved, and reviewed in the cloud operating model.

## Key Practices

- maintain an ownership model
- review security policies regularly
- track control exceptions centrally
- tie security decisions to risk and business impact
- ensure audit-ready evidence is available
- make escalation paths explicit
- keep policy review notes current

## Common Risks

- unclear decision ownership
- stale policy mappings
- inconsistent evidence collection
- poor exception tracking

## Evidence To Collect

- governance charter
- ownership model
- exception log
- policy review records
- security review outcomes

## Why It Matters

Security governance works best when the organization can answer who decided, why they decided it, and what evidence supports the decision.

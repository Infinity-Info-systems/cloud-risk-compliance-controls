# Compliance Risk Register

## Purpose

Track compliance-related risks, exceptions, and remediation actions.

## Suggested Fields

- requirement
- mapped control
- owner
- evidence
- gap description
- remediation status
- review cadence
- risk rating

## Example Risks

- incomplete evidence for periodic access review
- delayed policy updates after regulatory change
- unsupported exception without expiration date
- control mapping missing for a critical requirement

## Register Guidance

This register should act as the bridge between compliance language and the actual control work that needs to happen.
